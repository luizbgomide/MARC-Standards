FIELD DEFINITION AND SCOPE

Fifteen character positions (00-14) that contain positionally-defined, coded information about the record as a whole or about special community information aspects of the item being recorded. Each character position must contain either a defined code or a fill character ( | ). Code n (Not applicable), when it is defined for a data element, indicates that the character position definition does not apply.

These coded data elements are potentially useful for retrieval and data management purposes. A fill character may be used in a particular character position when the creator of the record has decided not to attempt to supply a defined code.

GUIDELINES FOR APPLYING CONTENT DESIGNATORS

■ CHARACTER POSITIONS

00-05 - Date entered on file
Computer-generated, six-character numeric stringthat indicates the date the MARC record was created. The date is recorded in the pattern yymmdd.
Use yy for the year, mm for the month, and dd for the day). The date entered on file in 008/00-05 is never changed. The date and time of latest transaction information in field 005 changes each time a transaction is made to the record. The latest transaction information enables an organization handling more than one version of a record to identify the most current version.
008/00-05 930819
[The record was entered into the system on August 19, 1993.]

12-14 - Language
Three-character MARC code that indicates the language associated with the community information entity. Code from: MARC Code List for Languages. When more than one language is associated with the entity, all appropriate codes are given in field 041 (Language Code) and the predominant language code is recorded in 008/12-14. Language information may be described textually in field 546 (Language Note).
When there is more than one language, the code recorded in 008/12-14 must always be the same as the first code in field 041, subfield $a. When there is only one language associated with the community information entity, the code for that language is recorded. If there is no predominant language associated with the community information entity, the codes are recorded in English alphabetical order in field 041, and the first one is recorded in 008/12-14.

INPUT CONVENTIONS

System-Generated Data Elements - 008/00-05 (Date entered on file) is usually system generated. System generation of other 008 data elements depends upon the capabilities of an individual system.
Capitalization - Alphabetic codes are input in lowercase.
