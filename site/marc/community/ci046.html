FIELD DEFINITION AND SCOPE

Coded date information that is potentially useful for retrieval and data management purposes. If none of these types of dates is applicable to the record in hand, the field is not used.

GUIDELINES FOR APPLYING CONTENT DESIGNATORS

■ INDICATORS

  • Both indicator positions are undefined; each contains a blank (#).

■ SUBFIELD CODES

$f - Action date
Date the record needs to be looked at again.
$g - Purge date
Date the information should be removed from the system as it is no longer relevant.
$h - Beginning date of event or program
$i - Ending date of event or program
$j - Date resource modified
Date on which a resource has been modified. Dates in subfield $j usually relate to Web pages and other electronic media that routinely contain the date of last update.
$k - Beginning or single date created
Beginning date of a date range on which a resource has been created, when it is not more appropriately recorded in another field. Dates contained in subfield $k may not be coded elsewhere in the formats.
$l - Ending date created
Ending date of a date range on which a resource has been created.
$m - Beginning of date valid
Beginning of the date span for the validity of a resource.
$n - End of date valid
Ending of the date span for the validity of a resource.
$2 - Source of date
Code that identifies the source of the date scheme used in the field. Code from: Date and Time Scheme Source Codes.
$6 - Linkage
See the description of this subfield in Appendix A: Control Subfields.
$8 - Field link and sequence number
See the description of this subfield in Appendix A: Control Subfields.

INPUT CONVENTIONS

Length of date - When part of a date is unknown, missing digits are represented by the character u.

CONTENT DESIGNATOR HISTORY

In 2002, field 046 was redefined as a repeatable field.

Field 004 - Coded Dates Fixed Field [OBSOLETE, 1998]
Due to a conflict with a differently defined field 004 in the MARC 21 Format for Holdings Data, field 004 was made obsolete in favor of four special subfields in field 046, defined in 1998. Subfield $f (Action date) was defined for information originally coded in 004/00-07. Subfield $g (Purge date) was defined for information originally coded in 004/08-15. Subfield $h (Beginning date of event or program) was defined for information originally coded in 004/16-23. Subfield $i (Ending date of event or program) was defined for information originally coded in 004/24-31. The restructuring of the field 004 positionally defined elements into separately delimited subfields eliminated the need to encode more dates than were actually needed in the record.
