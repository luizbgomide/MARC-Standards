Introdução

MARC 21 Bibliográfico
Outubro de 2006

Os cinco formatos de comunicação MARC 21, Formato MARC 21 para Dados Bibliográficos, Formato MARC 21 para Dados de Autoridade, Formato MARC 21 para Dados de Acervo, Formato MARC 21 para Dados de Classificação, e Formato MARC 21 para Informação de Comunidade, são padrões largamente utilizados para representação e troca de dados bibliográficos, de autoridade, de acervo, de classificação e de informação de comunidade em formato legível por máquina.

Um registro MARC é composto por três elementos: a estrutura de registro, a designação de conteúdo, e o conteúdo de dados do registro. A estrutura do registro é uma implementação do padrão internacional Format for Information Exchange (ISO 2709) e sua contraparte estadunidense Bibliographic Information Interchange (ANSI/NISO Z39.2). A designação de conteúdo -- os códigos e convenções estabelecidos para identificar e caracterizar os elementos de dados em um registro, possibilitando a manipulação destes dados -- é definido por cada um dos formatos MARC. O conteúdo dos elementos de dados que compõem um registro MARC é geralmente definido por padrões além destes formatos. São exemplos disso: International Standard Bibliographic Description (ISBD), Anglo-American Cataloguing Rules (AACR), Library of Congress Subject Headings (LCSH), ou outras regras de catalogação, tesauros de assunto, e esquemas de classificação utilizados pelas organizações que criam um registro. O conteúdo de certos elementos de dados codificados é definido nos formatos MARC (ex: o Líder, campo 007, campo 008).

O Formato MARC 21 para Dados Bibliográficos: Incluindo Diretrizes para Designação de Conteúdo define os códigos e convenções (etiquetas, indicadores, códigos de subcampo e valores codificados) que identificam os elementos de dados em registros bibliográficos MARC. Este documento é destinado para o uso do pessoal envolvido na criação e manutenção de registros bibliográficos, assim como daqueles envolvidos no desenvolvimento e manutenção de sistema para comunicação e processamento de registros bibliográficos. Esta documentação também está disponível online, incluindo uma versão concisa e uma lista simples de campos em: www.loc.gov/marc/.

ESCOPO DO FORMATO BIBLIOGRÁFICO

O Formato MARC 21 para Dados Bibliográficos foi criado para transporte de informações bibliográficas sobre materiais textuais impressos e manuscritos, arquivos de computador, mapas, músicas, recursos contínuos, materiais visuais e materiais mistos. Os dados bibliográficos comumente incluem títulos, nomes, assuntos, notas, dados de publicação e informações sobre a descrição física de um item. O formato bibliográfico contém elementos de dados para os seguintes tipos de material:

  • Livros (BK) - usado para materiais textuais impressos, eletrônicos, manuscritos e em microformas que são de natureza monográfica.
  • Recursos Contínuos (CR) - usado para materiais textuais impressos, eletrônicos, manuscritos e em microformas, distribuídos em partes com um padrão recorrente de publicação (ex: periódicos, jornais, anuários). (NOTA: Antes de 2002, os Recursos Contínuos (CR) eram referidos como Séries (SE)).
  • Arquivos de Computador (CF) - usado para softwares, dados numéricos, multimídia para computador, sistema online ou serviços. Outras classes de recursos eletrônicos são codificados por seu aspecto mais significativo. Os materiais podem ser de natureza monográfica ou seriada.
  • Mapas (MP) - usado para todos os tipos de materiais cartográficos impressos, eletrônicos, manuscritos e em microformas, incluindo atlas, mapas e globos. Os materiais podem ser de natureza monográfica ou seriada.
  • Músicas (MU) - usado para música impressa, eletrônica, manuscrita e em microformas, assim como gravações de som musicais e não musicais. Os materiais podem ser de natureza monográfica ou seriada.
  • Materiais Visuais (VM) - usado para mídias projetáveis, mídias não projetáveis, gráficos bidimensionais, artefatos tridimensionais, objetos naturais e kits. Os materiais podem ser de natureza monográfica ou seriada.
  • Materiais Mistos (MX) - usado principalmente para coleções de arquivos e de manuscritos de formas diversas de materiais. Os materiais podem ser de natureza monográfica ou seriada. (NOTA: Antes de 1994, os Materiais Mistos (MX) eram referidos como Material de Arquivos e Manuscrito (AM)).

Tipos de Registros Bibliográficos

Os registros bibliográficos MARC são diferenciados de todos os outros tipos de registros MARC por códigos específicos no Líder/06 (Tipo de registro), que identifica os seguintes tipos de registros bibliográficos.

Material textual Gravação de som não musical
Material textual manuscrito Gravação de som musical
Arquivo de computador Mídia projetável
Material cartográfico Gráfico bidimensional não projetável
Material cartográfico manuscrito Artefato tridimensional ou objetos naturais
Notação musical Kit
Música manuscrita Material Misto

Microformas, sejam originais ou reproduções, não são identificados como um tipo especial de registro. O aspecto de microforma é secundário ao tipo do material ao qual o item original pertence (ex: livro). O mesmo é vale para Arquivos de Computador em que o aspecto de arquivo de computador é secundário; no entanto, certas categorias de recursos eletrônicos são codificadas como Arquivos do Computador.

COMPONENTES DE REGISTROS BIBLIOGRÁFICOS

Descrição das Partes do Registro

Um registro bibliográfico MARC consiste de três componentes principais: o Líder, o Diretório e os campos variáveis. As seguintes informações resumem a estrutura de um registro MARC separado. Mais detalhes estão disponíveis em MARC 21 Specifications for Record Structure, Character Sets, and Exchange Media.

  • Líder - Elementos de dados que fornecem primariamente informações para o processamento do registro. Os elementos de dados contêm números ou valores codificados e são identificados pela posição relativa de caractere. O Líder tem tamanho fixo de 24 posições de caracteres e é o primeiro campo de um registro MARC.
  • Diretório - Uma série de referências que contêm a etiqueta, comprimento, e localização inicial de cada campo de variável dentro de um registro. Cada entrada tem 12 posições de caracteres de comprimento. As entradas do diretório para campos de controle variáveis aparecem primeiro, seguidas pelas etiquetas de campo em ordem numérica crescente. As entradas para campos de dados de variáveis seguem, organizadas em ordem ascendente de acordo com o primeiro caractere da etiqueta. A sequência armazenada dos campos de dados de variável em um registro não necessariamente corresponde à ordem das entradas do Diretório correspondente. Etiquetas duplicadas são diferenciadas apenas pela localização dos respectivos campos dentro do registro. O Diretório termina com um caractere terminador de campo (ASCII 1E hex).
  • Campos Variáveis - Os dados em um registro bibliográfico MARC estão organizados em campos de variáveis, cada um identificado por uma etiqueta numéria de três caracteres que é armazenada na entrada de Diretório para o campo. Cada campo termina com um caractere terminador de campo. O último campo variável em um registro termina com ambos, um terminador de campo e um terminador de registro (ASCII 1D hex). Existem dois tipos de campos variáveis.
  • Campos de controle variáveis - Os campos 00X. Estes campos são identificados por uma etiqueta de campo no Diretório, mas eles não contêm posições de indicador nem códigos de subcampo. Os campos de controle de variáveis são estruturalmente diferentes dos campos de dados de variáveis. Eles podem conter um único elemento de dados ou uma série de elementos de dados de comprimento fixo identificados pela posição de caractere relativa.
  • Campos de dados variáveis - Os campos de variáveis restantes definidos no formato. Além de serem identificados por uma etiqueta de campo no Diretório, os campos de dados variáveis contém duas posições de indicadores armazenadas no início de cada campo e um código de subcampo de dois caracteres precede cada elemento de dados dentro do campo.
  • Os campos de dados variáveis são agrupados em blocos conforme o primeiro caractere da etiqueta, que, com algumas exceções, identifica a função do dado dentro do registro. O tipo de informação no campo é identificado pelo restante da etiqueta.
0XX Informação de controle, números de identificação e classificação, etc.
1XX Entradas principais
2XX Títulos e parágrafo do título (título, edição, imprenta)
3XX Descrição física, etc.
4XX Indicações de série
5XX Notas
6XX Campos de acesso por assunto
7XX Entradas adicionais além de assunto ou série; campos ligados
8XX Entradas adicionais de série, acervo, etc.
9XX Reservado para implementação local

Dentro dos blocos 1XX, 4XX, 6XX, 7XX e 8XX blocos, certos paralelos da designação de conteúdo geralmente são preservados. Os seguintes significados, com algumas exceções, são dados aos dois últimos caracteres da etiqueta dos campos:

X00 Nomes pessoais X40 Títulos bibliográficos
X10 Nomes corporativos X50 Termos tópicos
X11 Nomes de eventos X51 Nomes geográficos
X30 Títulos uniformes          

Dentro dos campos de dados de variáveis, estes dois tipos de designação de conteúdo são usadas:

  • Posições de indicadores - As primeiras duas posições de caracteres nos campos de dados variáveis que contêm valores que interpretam ou complementam os dados encontrados no campo. Os valores indicadores são interpretados de forma independente, ou seja, não é atribuído um significado aos dois indicadores em conjunto. Os valores dos indicadores podem ser caracteres alfabéticos em minúsculas ou numéricos. Um espaço em branco (ASCII SPACE), representado neste documento como um #, é usado em uma posição de indicador indefinida. Em uma posição de indicador definida, um espaço em branco pode ter um significado específico ou significar nenhuma informação fornecida.
  • Códigos de subcampos - Dois caracteres que diferenciam os elementos de dados dentro de um campo que requer manipulação separada. Um código de subcampo consiste de um delimitador (ASCII 1F hex), representado neste documento como um $, seguido de um identificador de elemento de dados. Os identificadores de elementos de dados podem ser caracteres alfabéticos em minúsculas ou numéricos. Códigos de subcampo são definidos de forma independente para cada campo; no entanto, significados paralelos são preservados sempre que possível (ex: nos campos de Nomes Pessoais de 100, 400, e 600). Códigos de subcampo são definidos para fins de identificação, e não de arranjo. A ordem dos subcampos é geralmente especificada pelos padrões para o conteúdo dos dados, como as regras de catalogação.

Registros Bibliográficos com Múltiplos Sistemas de Escrita

Um registro bibliográfico MARC pode conter dados em múltiplos sistemas de escrita. Um sistema de escrita pode ser considerado a escrita principal dos conteúdos de dados do registro, mesmo que outras escritas também sejam usadas para conteúdos de dados. Modelos gerais para dados com múltiplos sistemas de escrita são descritos no Apêndice D juntamente com vários exemplos.

Repetibilidade de Campos e Subcampos

Teoricamente, todos os campos e subcampos podem ser repetidos. No entanto, a natureza dos dados muitas vezes impede a repetição. Por exemplo, um registro bibliográfico pode conter apenas um campo de entrada principal 1XX; um campo 100 pode conter apenas um subcampo $a (Nome pessoal), mas pode conter mais de um subcampo $c (Títulos e outras palavras associadas com um nome). A repetibilidade ou não de cada campo e subcampo é especificada no formato.

Ligação de Campos

Os campos no registro podem ser ligados de forma especial utilizando uma técnica de ligação de campos que geralmente é aplicável. A técnica depende da sintaxe dos dados no subcampo $8 (Ligação de campo e número de sequência) para identificar os campos ligados. A estrutura e sintaxe para o subcampo de ligação de campo e número de sequência são descritos no Apêndice A.

Caractere de Preenchimento e Valores Relacionados

Um caractere de preenchimento (ASCII 7C hex), representado nesse documento por uma barra vertical (|), pode ser usado em registros bibliográficos nos campos 006, 007, 008, e no subcampo $7 do campo 533 (Nota de reprodução) e nos campos de entrada ligada (760-787). Um caractere de preenchimento não pode ser usado no líder, ou em etiquetas, indicadores ou códigos de subcampo. O uso do caractere de preenchimento em registros que contribuem para uma base de dados nacional também pode depender dos requisitos nacionais especificados para cada elemento de dados. A presença de um caractere de preenchimento em um registro bibliográfico indica que o formato especifica um código para ser usado, mas o criador do registro decidiu não tentar fornecer um código.

O código u (Desconhecido ou não especificado), quando ele for definido, indica que o criador do registro tentou fornecer um código, mas não pode determinar qual deveria o código apropriado.

O código n (Não aplicável) é definido em muitas posições codificadas para indicar que a característica definida pela posição não era aplicável ao tipo específico de item ou ao tipo do registro.

Constantes de Exibição

Uma constante de exibição é uma convenção de termo, frase, espaçamento ou pontuação, que pode ser gerada pelo sistema para produzir uma apresentação visual mais significativa para o usuário dos dados em um registro. No formato bibliográfico, certas etiquetas de campos (ex: campo 770, Entrada de Publicação Suplementar/Especial), indicadores (ex: campo 511, indicador 1, controlador de constante de exibição), e códigos de subcampo (ex: ou subcampos de subdivisão de assunto $x, $y e $z em uma entrada secundária de assunto), pode ser usado para gerar convenções de termos, frases, e/ou espaçamento ou pontuação específicos para a exibição do registro. O uso das constantes de exibição é determinado por cada organização ou sistema. Exemplo de constantes de exibição são fornecidos sob Convenções de Uso nas descrições de campos.

Responsabilidade de Conteúdo do Registro

O Formato MARC 21 para Dados Bibliográficos funciona como um veículo para dados bibliográficos de todos os tipos e de qualquer organização. Em geral, a responsabilidade pelo conteúdo de dados, designação de conteúdo, e a transcrição de informações bibliográficas dentro de um registro pode ser determinada através do campo indicado na seção partes responsáveis abaixo. No entanto, o conteúdo de certos elementos de dados é restrito quando o elemento for atribuído por agência ou um elemento de dados de lista controlada.

Partes Responsáveis

Em registros não modificados, a organização identificada como a fonte original de catalogação em 008/39 e/ou 040 $a é responsável pelo conteúdo do registro. A organização identificada como a agência de transcrição no campo 040 $c é responsável pela designação de conteúdo e transcrição dos dados.

Nos registros modificados, as organizações identificadas em 040 $a e $d (Agência modificadora) são coletivamente responsáveis pelo conteúdo do registro. As organizações identificadas como agências transcritoras ou modificadores no campo 040 $c e $d são coletivamente responsáveis pela designação de conteúdo e transcrição dos dados.

Elementos de Dados Atribuídos por Agência

Um elemento de dado atribuído por agência é aquele cujo conteúdo é determinado por uma agência designada e é de responsabilidade desta agência, ex: campo 222 (Título Chave) que é responsabilidade de um Centro ISSN. Embora geralmente seja inserido pela agência designada, ele pode ser transcrito por outra organização.

Elementos de Dados de Lista Controlada

Certos elementos de dados contém dados de listas controladas mantidas por agências designadas, ex: a Lista de Códigos MARC para Áreas Geográficas no campo 043 (Código de Área Geográfica). Estes elementos são indicados nos níveis de campo ou subcampo no MARC 21 e apenas valores das listas designadas podem ser usados. Se for desejável uma alteração ou adição para uma lista, a agência mantenedora da lista deve ser consultada.

Requisitos de Nível de Registro

Grupos de usuários podem ter requisitos de registro de nível completo e nível mínimo, para promover consistência entre as agências de catalogação. Estes requisitos devem ser amplamente divulgados para que todos os possíveis parceiros de intercâmbio tenham ciência.

ORGANIZAÇÃO DESTE DOCUMENTO

Main Parts

MARC 21 Format for Bibliographic Data consists of a Summary Statement of Content Designators followed by a detailed presentation of each content designator. Following the descriptions of the Leader and the Directory, the detailed presentations for each variable field are arranged in field tag order.

Appendixes to this document provide information about several control subfields (Appendix A), full level record examples (Appendix B), minimal level record examples (Appendix C), multiscript record examples (Appendix D), an alphabetical listing of ambiguous headings with suggested tagging (Appendix E), a multilingual list of initial definite and indefinite articles (Appendix F), lists of changes to the format since the last edition or update (Appendix G), a listing of several Canadian and American local fields (Appendix H) and a list of organization code sources (Appendix I).

General Information Sections

To avoid repetition, general information sections are provided for groups of fields with similar characteristics. These general information sections provide instructions for the content designators that are common to each field in the group. The description for each of the fields refers back to the general information section for that group and to any related general information section.

For example, the X00 Personal Name--General Information section is provided for personal name headings. This general information section provides instructions for the content designators that are common to each type of personal heading whether it is used in a 100 main entry field, a 600 subject access field, an 700 added entry field, or a 800 series added entry field. The description for each of the individual fields (i.e., 100, 600, 700, 800) refers back to the X00 general information section.

Components of the Detailed Descriptions

A detailed description may consist of six parts: content designator listing; character position or field definition and scope; guidelines for applying content designators, with examples; input conventions; and content designator history.

The content designator listing area presents

  • the name of the content designator
  • the repeatability/nonrepeatability code (R or NR)

For the Leader and the variable control fields, the list gives the name of the character position and any defined coded value. For the variable data fields, the list gives the name of the indicator positions and any defined coded values and the name of the subfield codes and any defined coded values. For all variable fields, the repeatability/nonrepeatability code at the field level specifies whether the field may be repeated in a record. For variable data fields, the subfield repeatability code specifies whether the subfield code may be repeated within a field.

The character position or field definition and scope area describes the contents of the character position or field, the appropriateness of its use in specific kinds of bibliographic records, and gives other information, such as field repeatability in particular circumstances, the use of required lists or rules in formulating the data, etc.

The guidelines for applying content designators area describes the content and scope of each coded value, indicator, or subfield code and gives other information, such as appropriateness for use and repeatability in certain circumstances. The coded values and subfield codes in this area are presented in alphabetical/numerical order. Examples showing the use of the content designator are provided immediately following the description. The examples illustrate the application of specific MARC content designation. The data may be fuller or less full than would be used in actual cataloging practice. Most of the examples reflect the application of AACR 2 and ISBD; however, some reflect a generalized MARC application. Other examples illustrate specific points.

The input conventions area provides general guidance for the application of the content designators, and for such things as punctuation, spacing, and the use of display constants. The punctuation used within a field is generally dictated by descriptive cataloging or subject heading system/thesaurus rules. The input conventions clarify MARC punctuation practices especially with respect to final punctuation. In the discussion of punctuation practices, mark of punctuation is a period (.), a question mark (?), an exclamation mark (!), or a hyphen (-).

The content designator history area provides a record of significant content designator changes. The types of changes that are included are:

  • newly defined content designators that impact on coding consistency within a file, e.g., the location of a meeting entered under a corporate body was not separately subfield coded in the X10 fields prior to the definition of subfield $c in 1980.
  • redefined codes and values, e.g., in X10 fields, both the number and the name of a part/section of a work were contained in subfield $p prior to the redefinition of subfield $p for only the name of a part/section of a work in 1979.
  • changes in codes and values for consistency across MARC specifications, e.g., in 008/24-27 for the books specifications, code f (Handbooks) was identified by code h prior to 1979.
  • changes in repeatability when it impacts on file consistency, e.g., field 020 was not repeatable prior to 1977; multiple ISBN data were contained in repeatable subfields $a, $b, and $c.
  • restructuring, e.g., field 007 (Physical Description Fixed Field) in 1980.
  • obsolete content designation, e.g., subfield $b (Number) in the X11 fields when subfield $n was redefined to include meeting numbers in 1979. Obsolete content designation formerly defined in only one format are designated [USMARC only] or [CAN/MARC only]; unmarked items were defined in both formats.

In the MARC formats, a distinction is made between obsolete and deleted content designators. An obsolete content designator is one that may have been used in MARC records and that may continue to appear in records created prior to the date it was made obsolete. Obsolete content designators are not used in new records. An obsolete content designator is recorded in the Content Designator History area. Content designation instructions are provided for retrospective conversion of records having data elements that would have been identified by the obsolete content designator. A deleted content designator is not recorded in the history area and will no longer appear anywhere in the MARC documentation. A content designator that had been reserved in MARC but has not been defined, or one that had been defined but is known with near certainty not to have been used, may be deleted from the format. A deleted content designator is available for redefinition in a format. A deleted content designator is available for redefinition in a format.

Typographical Conventions

Throughout this document, the following typographical conventions are used:

  • 0 - The graphic 0 represents the digit zero in tags, fixed-position character positions, indicator positions, and other places numerics are used. This character must be distinguished from an uppercase letter O in examples or text.
  • # - The graphic symbol # is used for a blank in coded fields and in other special situations where the existence of the character blank might be ambiguous. (In most textual examples, the blank is represented in the conventional way, by the absence of a character.)
  • $ - The graphic symbol $ is used for the delimiter portion of a subfield code. Within the text, subfield codes are referred to as subfield $a, for example.
  • / - Specific character positions of the Leader, Directory, field 007, field 008 are expressed using a slash and the number of the character position, e.g., Leader/06, 007/00, 008/12.
  • 1 - The graphic 1 represents the digit one (hex 31). This character must be distinguished from a lowercase roman alphabet letter l (hex 6C) and uppercase alphabetic letter I (hex 4C) in examples or text.
  • | - The graphic | represents a fill character in MARC examples. When this mark appears in the left margin, it indicates areas of the text of this document where changes have been made.

DOCUMENTAÇÃO SUPLANTADA

Este documento substitui as edições mais recentes do USMARC Format for Bibliographic Data e do Canadian MARC Communication Format for Bibliographic Data. Com o alinhamento destes formatos em 1997, uma única edição do formato pode ser publicada.

Documentação do USMARC

MARC 21 substitui a edição de 1994 do USMARC Format for Bibliographic Data: Including Guidelines for Content Designation e atualizações n. 1 (março de 1995), n. 2 (março de 1996) e n. 3 (julho de 1997) para essa edição. Com a atualização n. 3, os formatos USMARC e CAN/MARC tiveram especificações de formato idênticas. A edição de 1994 substituiu a edição de 1988 e as suas quatro atualizações publicadas entre 1988 e 1991. A edição de 1988 incorporou o texto base da edição de 1980 publicada sob o título MARC Formats for Bibliographic Data (MFBD) e as 15 atualizações dessa edição publicadas entre 1980 e 1987. A edição de 1980 era uma compilação de documentos de formato MARC separados elaborados para diferentes tipos de materiais e publicados entre 1969 e 1977.

Documentação do CAN/MARC

O MARC 21 substitui a edição de 1994 do Canadian MARC Communication Format for Bibliographic Data e suas seis atualizações publicadas entre 1994 e 1998. A atualização seis continha as mudanças para CAN/MARC necessárias para se alinhar com o USMARC. Com esta atualização, as especificações contidas nos formatos CAN/MARC e USMARC são idênticas. A edição de 1994 substituiu a edição de 1988 e suas sete atualizações publicadas entre 1988 e 1991. A edição de 1988 foi produzida pela amálgama de formatos de CAN/MARC emitidos separadamente para monografias e séries.
